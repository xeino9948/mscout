# 2026-03-12

## Monorepo 改造完成

将 music-online-status-check 项目从前后端分离结构改造为 pnpm + Turborepo monorepo。

### 改造内容
- **目录结构**：`frontend/` → `apps/web/`，`server/` → `apps/server/`，新增 `packages/shared/`
- **包命名**：`@mscout/web`、`@mscout/server`、`@mscout/shared`
- **共享类型**：前后端共享的业务类型（SearchQuery, SongInfo, QualityInfo 等）抽取到 `@mscout/shared`
- **Hono RPC**：后端改为链式写法，从 `app.ts` 导出 `AppType`；前端用 `hc<AppType>` 替代手写 fetch，实现端到端类型安全
- **双模式入口**：`index.ts` 支持 Node.js 本地开发 + 边缘运行时导出
- **部署**：创建了 `wrangler.toml` 配置 Cloudflare Workers

### 关键决策
- server 的 `exports` 指向 `app.ts`（不是 `index.ts`），避免前端 tsc 检查 Node.js 专有代码
- shared 包使用 JIT 策略（直接导出 TypeScript），不需要编译步骤
- 添加了 `@hono/zod-validator` + `zod` 做搜索接口的请求验证
